# Support Guide

Thank you for using Kush Secure Protocol (KSP)! This document outlines the channels available for getting help, asking questions, or discussing features.

---

## 💬 Community Support

For general questions, configuration help, and architecture discussions:

* **GitHub Discussions**: The primary forum for the community. Use this for general questions, troubleshooting, and proposing changes.
  * 👉 [Ask a Question / Open a Thread](https://github.com/Kush2272/ksp/discussions)
* **GitHub Issues**: Please use issues *only* for filing actionable bug reports or concrete feature requests.
  * Before filing, search existing issues to see if it has already been reported.

---

## 🛡️ Security Support

If you suspect or discover a security vulnerability in KSP:

* **Do NOT open a public issue.**
* Refer to our [Security Policy](SECURITY.md) for instructions on how to submit a secure vulnerability report.
* Contact email: **kush@example.com**

---

## 📋 Response Commitments

As an open-source project managed by individual maintainers, support is provided on a best-effort basis:

| Support Channel | Expected Initial Response | Resolution Goal |
| :--- | :--- | :--- |
| **GitHub Discussions** | 3-5 days | Community-driven |
| **GitHub Bug Issues** | 48-72 hours | Variable (severity-dependent) |
| **Security Reports** | 24-48 hours | Priority fix & advisory release |
