1. ClientHello ▶
Client sends version list, capabilities, client_random, and ephemeral X25519 public key.
2. ServerHello ▶
Server selects version and cipher suite, sends server_random, server public key, and session UUID.
3. Certificate ▶
Server sends Ed25519 Certificate along with signature binding public keys to key exchange.
4. ECDH Shared Secret ▶
Both calculate the X25519 shared secret. Reject all-zeros week key outputs.
5. HKDF Key Expansion ▶
Session keys are derived: client_write_key, server_write_key, client_write_iv, server_write_iv.
6. Encrypted Auth ▶
Credentials (API key / Token) are encrypted under DerivedKeys and exchanged.
7. HandshakeFinish ▶
Both verify Finished transcript HMAC to prevent cipher downgrade attacks.
